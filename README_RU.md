# Minecraft Чит Мод

Fabric мод для Minecraft 1.21.1 с различными чит-функциями и меню настроек в игре.

## Возможности

1. **Без урона** - Игрок не получает урона из любых источников
2. **Бесконечное дыхание** - Бесконечное дыхание под водой
3. **Максимальный размер инвентаря** - Автоматически заполняет стаки предметов до максимума
4. **Убийство с одного удара** - Убивайте любого моба одним ударом
5. **Максимальный опыт** - Автоматически устанавливает уровень опыта на 10000
6. **Неразрушимые инструменты** - Инструменты и предметы никогда не ломаются

## Настройка в игре

Нажмите клавишу **U** (по умолчанию) чтобы открыть меню настроек мода. Вы можете включать и выключать любую функцию без перезапуска игры. Настройки автоматически сохраняются в `config/cheatmod.json`.

### Как использовать меню:

1. Запустите Minecraft с установленным модом
2. Находясь в игре, нажмите клавишу **U**
3. Откроется меню с кнопками для каждой функции
4. Нажмите на кнопку, чтобы включить (зеленый ON) или выключить (красный OFF) функцию
5. Нажмите "Done" для закрытия меню
6. Все изменения сохраняются автоматически

### Изменение клавиши открытия меню:

1. В главном меню Minecraft перейдите в "Настройки" → "Управление"
2. Найдите категорию "Cheat Mod" / "Чит Мод"
3. Найдите "Open Cheat Menu" / "Открыть меню читов"
4. Нажмите на кнопку и назначьте новую клавишу

## Установка

1. Установите Minecraft 1.21.1
2. Установите Fabric Loader 0.16.5 или выше
3. Скачайте Fabric API с [CurseForge](https://www.curseforge.com/minecraft/mc-mods/fabric-api)
4. Поместите JAR файл мода в папку `.minecraft/mods`
5. Запустите Minecraft с профилем Fabric

## Сборка из исходного кода

### Требования
- Java 21 или выше
- Gradle (включен через wrapper)

### Шаги сборки

```bash
chmod +x ./gradlew
./gradlew build
```

Скомпилированный JAR файл мода будет находиться в папке `build/libs/`.

## GitHub Actions

Этот мод включает автоматическую сборку и релиз через GitHub Actions:

- **Автоматическая сборка**: Каждый push в ветки main/master запускает сборку
- **Создание релиза**: Создайте git тег (например, `v1.0.0`) для автоматического создания релиза с скомпилированным JAR файлом

### Как создать релиз:

#### Через командную строку:
```bash
git tag v1.0.0
git push origin v1.0.0
```

#### Через веб-интерфейс GitHub:
1. Перейдите на главную страницу вашего репозитория
2. Нажмите **Releases** → **Create a new release**
3. Нажмите **Choose a tag** → введите имя тега (например, `v1.0.1`)
4. Выберите **+ Create new tag: v1.0.1 on publish**
5. Заполните название и описание релиза
6. Нажмите **Publish release**

После этого GitHub Actions автоматически:
- Соберет проект
- Создаст релиз
- Прикрепит JAR файл к релизу

См. [GITHUB_ACTIONS.md](GITHUB_ACTIONS.md) для подробных инструкций.
