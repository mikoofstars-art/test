# GitHub Actions для сборки и публикации мода

Проект настроен с автоматической сборкой и публикацией через GitHub Actions.

## Автоматическая сборка

Файл `.github/workflows/build.yml` автоматически собирает мод при каждом push в ветки main или master.

### Что происходит при сборке:

1. Код загружается из репозитория
2. Устанавливается Java 21
3. Выполняется команда `./gradlew build`
4. Готовый JAR файл загружается как артефакт

### Как скачать собранный мод:

1. Перейдите в раздел **Actions** вашего репозитория на GitHub
2. Выберите последний успешный workflow run
3. Скачайте артефакт `cheatmod-artifacts`
4. Распакуйте ZIP файл и получите готовый JAR

## Создание релиза

Файл `.github/workflows/release.yml` автоматически создает релиз при создании тега версии.

### Как создать релиз:

#### Вариант 1: Через командную строку

```bash
git tag v1.0.0
git push origin v1.0.0
```

#### Вариант 2: Через веб-интерфейс GitHub

1. Перейдите на главную страницу репозитория
2. Нажмите **Releases** → **Create a new release**
3. Нажмите **Choose a tag** → введите имя тега (например, `v1.0.1`)
4. Выберите **+ Create new tag: v1.0.1 on publish**
5. Заполните название и описание релиза
6. Нажмите **Publish release**

#### Вариант 3: Ручной запуск

1. Перейдите в раздел **Actions**
2. Выберите workflow **Create Release**
3. Нажмите **Run workflow**
4. Выберите ветку и нажмите **Run workflow**

### Что происходит при создании релиза:

1. Код собирается с помощью Gradle
2. Создается новый Release на GitHub
3. JAR файл автоматически прикрепляется к релизу
4. Генерируются автоматические release notes

## Требования

- Java 21 установлена в CI окружении (настроено автоматически)
- Gradle wrapper включен в репозиторий
- Права на создание релизов (настроено через `GITHUB_TOKEN`)

## Структура артефактов

После сборки создается файл:
```
build/libs/cheatmod-1.0.0.jar
```

Этот файл нужно поместить в папку `.minecraft/mods/` для использования мода.
